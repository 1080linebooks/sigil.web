Find & Replace

— Searching In Code View —

Finding text and updating it is one of the most common tasks in Sigil – to look for words, to remove unwanted text in one file, or to change the formatting of specific items in the entire book. In fact, if you ever need to make lots of changes to your text you will almost certainly want to use Find & Replace since it allows you to quickly make a large number of edits across all your files.

Note

Most Find & Replace functions are only available in Code View. In fact, Book View only supports finding text in the current file. This chapter assumes you will use Find & Replace in Code View.

To open the Find & Replace dialog, use the menu option Search→Find/Replace: or the keyboard shortcut Ctrl+F:

Tip

Keyboard shortcuts exist for many find and replace operations – such as Ctrl+G to find again. See the Search menu for a list of shortcuts.

You can set various modes (Normal, Case-Sensitive, search some or all HTML files, search down, etc.) to modify the behavior of the actions you select (count, find, replace, etc.).

When open, the Find & Replace dialog is always positioned at the bottom of the editing windows for easy access.

Actions

The four buttons on the right are used to run your search commands.

Tip

After you run a find, replace or count, the text you entered in either the Find or Replace box is saved and is available for you to select later. Click the down-arrow to the right of the Find (or Replace) text box to show the list of items in the search histories.

Whenever you run a search, the results are shown in the Find & Replace status bar.

Find

Finds the next occurrence of your search term and highlights it.

To search for text, enter it in in the Find text box and press the Find button. Any text in the Replace box is ignored if you are just doing a Find.

Tip

If you select text and then use Ctrl+F to open or re-open the Find box, your selected text will be placed in the Find box.

Count All

Displays a count of the number of times the search term you entered in the Find text box occurs in the selected files.

Count All always counts from the start of a file.

Tip

Its a good idea to run Count All before doing a Replace to check the number of replacements are what you expect. And you should run it after a Replace All in case the particular search and replace created new matches.

Replace

Replaces one occurrence of your search term in your text at a time.

To replace text, enter the old text in the Find box, and the new text in the Replace box. Then select Replace.

The first time you do a Replace the first match will be highlighted. Subsequent Replaces will replace the highlighted matched text and then highlight the next match.

Tip

You can use the Undo icon or right-click in the editing window and select Undo to change the text back. But if you are replacing across files and your next match is on a different page, you need to switch back to the file containing the replaced text before selecting undo.

You can safely use Replace instead of Find if no text is selected since it will act just like Find until a match is highlighted.

Replace All

Replaces all text matching your search term with your replace term in all the selected files. This is done all at once automatically.

Note

You cannot undo a Replace All.

Replace All always replaces from the start of a file.

Tip

Test your find and replace strings using Replace a few times before using Replace All to avoid surprises.

Save your book before a Replace All and check it after the replace in case of issues.

Check if your mode is set for Current File or All HTML Files.

Mode – What to Search

You can control what type of text to search for using these options.

Normal

This mode will search for exactly what you type except that it will ignore case, e.g. “A” and “a” will be considered the same letter.

Case Sensitive

This mode will search for exactly what you type.

Regex

Regex is an abbreviation of “Regular Expression”. This mode allows you to search for patterns instead of exact words or characters.

For instance, you can search for “Page [0-9]+” which will find "Page" followed by one or more numbers.

Regex is very powerful and one of the main benefits of using Sigil to work with ebooks.

See the Regular Expressions chapter for more details about using regex.

Spell Check

In Spell Check mode, Find can be used to find the next misspelled word.

Tip

In Code View you can set your preferences to show misspelled words with a wavy red underline, but this means you have to scroll through the document to find each misspelled word. Using Spell Check mode in Find & Replace allows you to find misspelled words one by one.

To use Spell Check mode, make sure the Find text box is empty, then click Find to find the next misspelled word.

You can then edit the word in the text or use replace. If automatic spell checking is on you can also right-click the misspelled word to select an alternative.

Tip

You can use Count All to easily count the number of misspelled words.

You can also use a regex expression in the Find box if you need to find specific misspelled words or patterns.

Note

See the Spell Check chapter for more information about spell checking.

Mode – Where to Search

You can control which files to search with these options making it easy to search all or just some of your files.

Current File

This restricts your search to just the file you are viewing. You can search HTML files and Style files.

If your search reaches the end of the file it will automatically wrap around and search from the other end of the file. You will see a small picture of an arrow flash on the screen to indicate the search wrapped around.

All HTML Files

This searches in every HTML file.

When a search reaches the end or beginning of the book it will automatically move to the next or previous file (in the order listed in the book browser), and it will loop around when all files have been searched.

Selected HTML Files

This searches only the HTML files that you selected in the Book Browser.

When a search reaches the end of a file it will automatically move to the next or previous selected file (in the order listed in the book browser), and it will loop around when all files have been searched.

Mode – Search Direction

You can control the direction of searching with these options.

Up

Searches from your position in the document upwards – finds the previous occurrence of your search.

Down

Searches from your position in the document downwards – finds the next occurrence of your search.
