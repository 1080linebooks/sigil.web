Table of Contents (TOC)

— A Map Of Your Book —

EPUB supports a Table of Contents (abbreviated as TOC) to help readers navigate their way around your book. The Table of Contents window in Sigil gives you a dynamic view of your TOC – you can view exactly what your TOC looks like, select entries to jump to, and generate new versions of your TOC automatically.

View the TOC

To open the TOC window use View→Table of Contents:

Tip

Click on an entry to jump to its location in the book.

Click on the arrow icons to expand or collapse entries with subsections.

Use the right-click context menu to select Collapse All or Expand All to quickly show or hide sub-chapters.

An EPUB Table of Contents is stored in the toc.ncx file, which is listed in the Book Browser. You can edit this information manually, but Sigil provides a way to generate its contents automatically.

Create Headings

In order to create a Table of Contents automatically you first need to define the headings or chapters in your book.

In Book View you can create the your headings by selecting the text you want to make into a heading, and then selecting the level of the heading you want from the Paragraph Format drop-down menu.

In Code View, use the standard h1, h2, etc. tags.

Generate TOC From Headings

You can use the Generate TOC from headings button to select the headings you want included in the book's Table of Contents (which is stored in the toc.ncx file).

Check any name that you want to be included in the Table of Contents, and uncheck any entry that you don't want included. You can use the TOC Items Only field to show or hide entries that will not be included in the TOC.

Tip

You can also use the Select headings to include drop-down menu to quickly select or unselect headings of a certain type, and then fine tune the selections manually.

If your chapter names do not use heading tags (h1, h2, etc.) as described above, then the chapter names will not be able to be used to generate a TOC automatically.

Note

When the TOC is auto-generated, Sigil will add ID names to every chapter if they don't already have one. If you copy a chapter heading to another part of the book, you may end up with duplicate heading ID names and you will need to edit these names.

If your heading is "near" the beginning of your HTML file, Sigil will link directly to the file and not to the heading. This is because on some ereaders linking directly to an element slows down the display of the TOC.

Warning

Generating the TOC automatically will overwrite any existing TOC in the toc.ncx file.

Generate Inline HTML TOC

You can use the button Generate TOC from headings to create an Inline HTML TOC for your EPUB. An inline TOC is just another HTML page in the EPUB that contains the Table of Contents in text format.

For most EPUB ereaders, the TOC that is automatically generated from Generate TOC from headings or stored in toc.ncx is used to display the TOC. However, there are times when an inline TOC is desired in an EPUB or when it is needed because the EPUB will be converted to another ebook format that does not use the TOC file.

Tip

If you are creating an EPUB you should leave out the inline HTML TOC file unless you have a clear reason to include it. If you need a TOC for a specific format you are converting to then you can include it just for that conversion.

The Generate Inline HTML TOC button will create a linked table of contents and overwrite any existing HTML file that has a guide semantic of TOC. If no such file exists it will create an HTML file called toc.html and mark its semantic type as TOC. If toc.html already exists it will be updated.

Tip

You can change the style of the inline TOC once you've generated it – but remember to do this each time you create the TOC.
