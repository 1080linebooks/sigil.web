Book View

— WYSIWYG Editing —

Book View (or WYSIWYG mode – what you see is what you get) is one of two ways you can edit your book in Sigil. Book View is what most people use when they start to use Sigil since its the closest to a standard word processor. Its a good way to make basic changes to your book when you aren't too concerned about the underlying HTML code. However, when you want more control over the format of your EPUB or to use Sigil's advanced features then you can use Code View. You can switch between Book View and Code View at any time.

To switch to Book View use View→Book View or its toolbar icon or keyboard shortcut:

Note

Certain features, like advanced Find & Replace and Spell Check are only available in Code View.

Book View will do its best to apply the style you intended, but occasionally it will guess wrong and you will need to reformat it. It may even be necessary to use Code View to adjust the formatting.

If you delete a blank line in Book View, you should click at the end of the previous paragraph then press the Delete key, or use the Backspace key – this is because you will lose the formatting of the following paragraph or header if you press Delete on the blank line.

Sometimes parts of the text are highlighted even though they are not selected. This is a bug in the Qt libraries used by Sigil and cannot be avoided.

If you edit some text and the style looks completely wrong, try switching to Code View and back to Book View as this may help refresh the screen.

If you change styles using stylesheets then you may need to save and re-open your book to see the new styles.

Tip

At the bottom of the Book View window you can see the HTML formatting tags that are applied to the text you have selected.

Toolbar Icons

In addition to the standard main toolbar icons like Undo, Copy and Paste, the Book View Editor has its own toolbar:

Details of each what each icon is used for is below:

  • Paste as plain text: standard paste of text
  • Paste from Word: removes certain styling from text copied from Word to avoid issues

  • B: make text bold
  • I: make text italic
  • U: make text underlined
  • abc: make text striked through
  • X2: make text subscript
  • X2: make text superscript, useful for footnotes (but harder to use on touch screens)
  • Erase: remove basic formatting (styles, classes but not ids) from selected text

  • Numbered List: change text into a list of items preceded by numbers
  • Bullet List: change text into a list of items preceded by dots
  • Outdent: remove an indent from the text
  • Indent: move text to the right
  • Block Quote: change text into a separate paragraph that may be treated as a quotation

  • Left Justify: align text to the left margin
  • Center: align text centered on the screen
  • Right Justify: align text to the right margin
  • Normal: reset to unjustified text

  • Left to Right: set the direction of text from left to right
  • Right to Left: set the direction of text from right to left

  • Link: create or edit a link for the selected text – see the Links and Footnotes chapter
  • Unlink: remove a link from the selected text
  • Anchor: create an anchor tag with an ID using the selected text – to use the text as the destination for a link

  • Table: insert simple tables, best used for data not for formatting text
  • Insert horizontal line: insert a line in the text
  • Insert special character: choose from special characters like Æ, ©, →, £, ½, É to insert

  • Format: contains formats to use for Chapter Headings (use Normal to change back to a standard paragraph and the erase icon to remove old formatting or style classes)

  • Text Color: set the color of the text
  • Background Color: set the color of the text background

Tip

Double click on an image to show the image's filename.

Customizing the Toolbar

The Book View editor uses the open-source CKEditor software. This is an extremely flexible editor that can be customized by advanced users.

Warning

Customizing the CKEditor settings is not supported, and it could result in some things breaking. If you are willing to customize the look of the Book View editor, then it may be better if you considering using Code View instead where you can control even more of what formats are used.

To modify the look and functionality of the Book View editor in Sigil you need to create a custom configuration file that Sigil will read when it starts. You can use this file to change what buttons are displayed, what menu items are shown for the editor, and what formats are applied when you select the icons or menu entries.

The official CKEditor documentation listing all the available configuration options is available at:

  • CKEditor Documentation

In Sigil, you need to create this file

ckeditor_config.js

and store it in Sigil's user directory (on linux this is '$HOME/.local/share/data/sigil-ebook/sigil/. You can find it by going to Spell Check in Preferences and selecting Open Dictionary Directory, then going up one level once the directory is open.

Here is a configuration file as an example:

CKEDITOR.editorConfig = function(config)
{
    // Toolbar example - commenting some icons out from default settings
    config.toolbarCanCollapse = false;
    config.toolbar = 'Full';
    config.toolbar_Full =
    [
        { name: 'clipboard', items : [ 'PasteText','PasteFromWord' ] },
        { name: 'basicstyles', items : [ 'Bold','Italic','Underline','Strike','Subscript','Superscript','-','RemoveFormat' ] },
        { name: 'format', items : [ 'NumberedList','BulletedList','-','Outdent','Indent','-','Blockquote','CreateDiv', ] },
        { name: 'align', items : ['JustifyLeft','JustifyCenter','JustifyRight','JustifyBlock' ] },
        //{ name: 'direction', items : [ 'BidiLtr','BidiRtl' ] },
        { name: 'links', items : [ 'Link','Unlink','Anchor' ] },
        { name: 'insert', items : [ 'Table','HorizontalRule','SpecialChar','PageBreak' ] },
        { name: 'styles', items : [ 'Styles','Format','Font','FontSize' ] },
        //{ name: 'colors', items : [ 'TextColor','BGColor' ] },
    ]

    // Size Menu
    config.fontSize_sizes = '80%/80%;90%/90%;100%/;110%/110%;120%/120%;1 em/1em; 2em/2em';

    config.font_names =
        'Arial/Arial, Helvetica, sans-serif;';

    // Styles Menu - class names must be defined in your CSS in order to be useful
    config.stylesSet = [
        { name : 'Strong', element : 'strong' },
        { name : 'Emphasis', element : 'em' },
        { name : 'Span Class Example', 
            element : 'span', 
            attributes : { 'class' : 'example' } },
    ];

};
