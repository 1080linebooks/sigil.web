Tips

When you have finished editing, you'll want to review your book to make sure you've covered everything. You'll soon build up your own approach, but here's a checklist to get you started:

  • Use Metadata Editor button to set your Title, Author, and Language. And if you want, also set the Copyright, Publication Date, Subject, Description, and ISBN if there is one.
  • Set your cover image and cover file (if you have one) as type Cover – see Cover Image.
  • Right-click on your first text file (where the book may open) and select Add Semantic→Text.
  • Regenerate your TOC, and re-create your HTML TOC if there is one.
  • Test your TOC links work as expected.
  • Test your links or footnotes work as expected.
  • Click on the Validate button to find errors then fix them – see Validation for typical error messages and what to do about them.
  • Check your book with the Spellcheck button , and consider creating a separate dictionary for your book if it has lots of unique words.
  • Link your stylesheets to all files that need them.
  • Check your stylesheets are valid using Tools→Validate Stylesheets with W3C.
  • Remove unused images, audio, or video files or unused style classes using Tool→Delete Unused...
  • Run Tools→Reports and look at the reports for anything unexpected (extra files, broken links, etc.)
  • If you want to use proper stylesheets, search for "sgc-" classes that might have been created by HTML Tidy, "<style>" tags used in inline CSS, or "style=" attributes used by inline styling and convert them CSS stylesheet files.
  • If you're really keen, search for and remove or replace text like &nbsp;, &quot;, <br />, empty paragraphs <p>&nbsp;</p>, empty <div></div> tags, empty attributes like src="", href="", or ="", change – to ndash, etc.. – unless of course you want to keep them. Especially after using Book View for that last quick update.
  • And if that isn't too much for you, you can even scan your book in Code View line by line for unexpected formatting, especially if you've used Book View to make updates.
  • Proofread on actual ereaders. Sigil can only give an approximation of what your book will look like. Despite EPUB being a standard, there are a lot of ereader devices and software they all have their own quirks and bugs.

And in case you missed it – test your book on actual devices and ereader software!
