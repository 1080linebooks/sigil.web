Tips

— Advice On Using Sigil —

This chapter is a list of tips from experienced users of Sigil.

Everyone uses Sigil differently but the tips may still be helpful to you. Keep in mind that Sigil is used by people making only a few occasional updates to books as well as people converting or cleaning up books professionally on a daily basis.

When you start to use Sigil more often you may find some of these suggestions useful:

  • Save often to allow automatic error checking and cleanup to run – to catch issues before they become larger.
  • Make backup copies of your file as you edit, especially before doing major changes.
  • Learn about stylesheets (CSS) and use them to control formatting instead of inline styles. This will make formatting your book much easier in the long run.
  • Use Code View for editing. Book View works great, especially for quick word corrections and some basic styling like headers, but if you want to format a lot of text or to control the exact styles used, then use Code View to avoid Book View using its own styles or inserting extra blank spaces, etc. And since many of Sigil's advanced functions are only available in Code View you should become familiar with it sooner rather than later.
  • Set Clean Source to Pretty Print instead of the default Tidy setting. Then exit and restart Sigil. This will help to clean up the spacing of code without changing styles or the code itself. Using Tidy can be helpful for a lot of people, but once you start editing your HTML code directly you won't want it creating inline style tags or making other changes. If you really don't want your code changed at all, then turn Clean Source off.
  • Keep the Well-Formed check turned on. If this is turned off then Sigil will automatically correct errors when you switch views or save – which could mean deleting some of your text. Leaving it on means you will be prompted to fix any errors.

When you have finished editing, consider following this checklist:

  • Validate to check for errors then fix them.
  • Spell Check, and consider creating a separate dictionary for your book if it has lots of unique words.
  • Set Title, Author, Language, Copyright and, if you want, Publication Date, Subject, Description, Copyright and ISBN if there is one.
  • Use Add Semantics to set your cover image and cover file (if you have one) as type Cover.
  • Set your first text file (where the book will open) as semantic type Text.
  • Test your text links or footnote links work as expected.
  • Regenerate your TOC or check that it's up to date and working, and regenerate your inline TOC if there is one.
  • Test your TOC links work as expected.
  • Link your stylesheets to all files that need them.
  • Search for and remove or update text like &nbsp;, &quot;, <br />, empty paragraphs <p>&nbsp;</p>, empty <div> tags, src="", href="", - to ndash, etc..
  • Search for sgc- classes, <style> tags or style= attributes and convert them CSS files.
  • Scan your book in Code View line by line for unexpected formatting, especially if you've used Book View to make updates.
  • Validate again.
  • Proofread on actual ereaders. Sigil can only give an approximation of what your book will look like. There are lots of ereaders and not all software or devices support all EPUB features and formatting in the same way.
Features you might have missed:
  • Ctrl-Space removes formatting.  In Code View it removes all tags/attributes for the selection, but leaves your text.  Select text, use Ctrl-Space, then use one of the heading buttons or Clip Editor clips to modify the selected text.
  • When using Clip Editor to paste in text you can have it add tags at the start and end of your selected text by using \1 to represent your text.
  • Hold the Ctrl key down while clicking on any Find & Replace or Search Editor action and your command will only be run on the Current File.  So you can leave it set to All HTML Files and just switch to the current file temporarily by using Ctrl.
  • Hold the Ctrl key down while pasting with Clip Editor and the code tags won't be removed when pasting.
